# モバイルレイアウト設計（iPhone / 320〜767px）

## 基本構造

```text
┌─────────────────────┐
│ ヘッダー（小、ページ名+接続状態）│ ← env(safe-area-inset-top)
├─────────────────────┤
│                     │
│  1 カラムコンテンツ           │
│  （カード / リスト）          │
│                     │
├─────────────────────┤
│ 概要 アプリ [操作] ログ ｼｽﾃﾑ │ ← 下部ナビ、高さ 56px+
└─────────────────────┘ ← env(safe-area-inset-bottom)
```

- サイドバーは表示しない。下部ナビ 5 項目固定。
- 中央「操作」ボタンはグローバル操作ボトムシートを開く:
  アプリ追加 / ターミナル / ファイルアップロード / ──区切り── / PC 再起動 / PC シャットダウン（電源系は視覚分離 + 赤系）
- タップ領域は最低 44×44px。主要ボタンは画面下半分（片手到達圏）に置く。

## Safe Area

```css
.bottom-nav { padding-bottom: env(safe-area-inset-bottom); }
.app-header { padding-top: env(safe-area-inset-top); }
```

`viewport-fit=cover` を meta viewport に指定。ノッチ / Dynamic Island / ホームインジケーターと重ねない。

## パターン

| 用途 | パターン |
|---|---|
| アプリ詳細・メトリクス詳細 | ボトムシート（ドラッグで閉じる、Esc/背景タップ対応） |
| アプリのその他操作 | カード右端 ⋯ またはロングプレス → ボトムシートメニュー |
| アプリ追加 | 全画面シート、3 ステップ |
| 確認（破壊的のみ） | 中央ダイアログ、明確な文言 + 赤ボタン |
| 完了通知 | トースト（下部ナビの上） |

## ページ別

- **概要**: メトリクスを 2×2 コンパクトカード（CPU/RAM/GPU/VRAM）+ 稼働時間・実行中アプリ数の行。
  グラフは初期 CPU/RAM のみ、他は展開。横スクロールは多用しない。
- **アプリ**: 1 カラムカード。常時表示 = アイコン / 名前 / 状態 / 稼働時間 / 主操作 1 個。
  タップで詳細ボトムシート（CPU/RAM/PID/ポート + ログ / 再起動 / 設定 / 削除）。
- **ログ**: 検索 + 一時停止のみ上部表示。他（DL / 削除 / 折り返し / フィルター）はボトムシート。仮想スクロール。
- **ターミナル（Phase 4）**: 全画面。visualViewport で高さ再計算。Ctrl/Alt/Tab/Esc/矢印の補助バー（横スクロール可）。
- **横向き**: ターミナル / リモートは横向き全画面対応。下部ナビは横向きでも維持（高さ縮小）。

## 通信・電池

- バックグラウンド（`visibilitychange`）でメトリクス購読を停止、復帰時に再開 + 最新スナップショット取得
- タブ復帰時に全 API を再取得しない（TanStack Query の staleTime を適切に設定）
