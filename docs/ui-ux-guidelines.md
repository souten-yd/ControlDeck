# UI/UX ガイドライン

コンセプト: **Minimal / Fast / Contextual / Progressive Disclosure / Touch Friendly / Keyboard Friendly**

## 原則

1. **2 ステップアクセス**: どの主要機能にも 2 操作以内で到達できる。3 階層以上のメニュー禁止。
2. **段階的情報開示**: 常時表示は要点のみ。詳細は展開パネル / ドロワー / ボトムシート / メニューで開く。
3. **主操作は 1 個**: 一覧アイテムには状態に応じた主操作ボタン 1 個 + その他メニュー（⋯）のみ。
   全操作ボタンの常時表示は禁止。
4. **破壊的操作の保護**: 削除・電源操作・強制終了のみ確認ダイアログ。通常操作はトースト + 状態変化で通知。
5. **赤の限定**: 赤色は破壊的操作と重大エラーのみに使用。

## ビジュアル

- 余白広め、線・枠を最小限。カード階層は最大 2 段。影は控えめ（sm まで）。
- 色は状態表現に限定: RUNNING=緑 / STOPPED=グレー / STARTING=青 / FAILED=赤 / DEGRADED=黄 / UNKNOWN=薄グレー。
  色 + アイコン + 文字を必ず併用（色覚対応）。
- 数値は `tabular-nums`。メトリクス数値が更新で揺れないよう幅を固定。
- ダーク / ライト / システム追従。テーマカラーとアプリ名は設定で変更可能（将来）。
- アニメーションは 100〜200ms、`prefers-reduced-motion` 対応。

## 状態表示

すべての非同期操作は Idle / Loading / Success / Error / Offline / Reconnecting を区別する。

- 起動操作 → 即座に STARTING 表示（楽観的 UI）→ WS/ポーリングで実状態へ収束
- WebSocket 切断時は最後の値を保持し「再接続中」バッジを表示、バックグラウンドで再接続
- エラーは「簡潔な原因 + 次の操作（再試行 / 詳細ログ）」。スタックトレースは出さない
- スケルトンローディング使用。レイアウトが跳ねないよう高さを予約

## ナビゲーション

- デスクトップ（≥1280px）: 左サイドバー（縮小可、縮小時はアイコン + ツールチップ）、上部バーに検索 / 接続状態 / ユーザーメニュー
- タブレット（768〜1279px）: サイドバーはアイコンのみ
- モバイル（<768px）: 下部ナビ 5 項目（概要 / アプリ / 操作 / ログ / システム）。中央「操作」はボトムシート
- コマンドパレット: Ctrl+K / Cmd+K（アプリ検索・起動停止・ページ移動）
- 電源操作はグローバル操作メニュー → 電源 → 確認、の 2 ステップ + 確認ダイアログ。常時露出させない

## 性能

- Monaco / React Flow / xterm.js / チャートは React.lazy で遅延ロード
- ログ・ファイル一覧は仮想スクロール
- メトリクス更新はセレクター経由で該当コンポーネントのみ再描画
- 非表示タブ・バックグラウンド時は購読を停止または低頻度化
- 一覧データは TanStack Query でキャッシュし、画面遷移で全体リロードしない
